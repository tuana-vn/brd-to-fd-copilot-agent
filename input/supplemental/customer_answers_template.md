# Customer / Domain Expert Answers

> Purpose: Please provide answers to the clarification questions below.  
> These answers will be used as supplemental evidence to update the Functional Design.  
> Please do not edit the question IDs. If a question cannot be answered now, write `Pending` and add the reason.

## Answering Rules

- Keep the original `QID` unchanged.
- Answer one question per row.
- Use clear and concise wording.
- If the answer depends on a condition, state the condition explicitly.
- If the source document is correct as-is, write `Confirmed as documented`.
- If the source document should be corrected, describe the correction.
- If the item is unsupported, write `Unsupported` and explain the expected handling.
- If the answer is unknown or requires another owner, write `Pending` and specify the owner or next action if known.
- Preserve exact names for commands, options, fields, states, messages, product names, and acronyms.

## Answer Status Values

Use one of these values in the `Answer Status` column:

- `Answered`
- `Partially answered`
- `Pending`
- `Not applicable`
- `Rejected / not required`
